The stories system is the core of your newsroom. Stories are fetched from Prezly via the SDK and rendered on the homepage, with each individual story available at its own URL. You can choose between two listing strategies — paginated navigation or infinite scroll — and switch between them with a single line change.

## Story listing

The homepage (`/`) renders the `Stories` module, which receives a `stories` array and a `pagination` object from `getHomepageServerSideProps`. By default it uses `PaginatedStories`, which renders the list alongside a `Pagination` component. The inner `StoriesList` component maps each story to a `<Link>` pointing to `/${story.slug}`.

```tsx
// modules/StoriesList/StoriesList.tsx
export function StoriesList({ stories }: Props) {
    return (
        <>
            {stories.map((story) => (
                <Link
                    key={story.uuid}
                    href={`/${story.slug}`}
                    style={{ display: 'block', marginBottom: 20 }}
                >
                    <span>{story.title}</span>
                </Link>
            ))}
        </>
    );
}
```

## Listing modes

You can choose between two listing modes by editing `modules/Stories/Stories.tsx`. The default is `PaginatedStories`. To switch to `InfiniteStories`, comment out `PaginatedStories` and uncomment the `InfiniteStories` line.

<Tabs>
  <Tab title="PaginatedStories (default)">
    ```tsx
    // modules/Stories/Stories.tsx
    import { PaginatedStories } from '../PaginatedStories';

    export function Stories({ stories, pagination }: Props) {
        return (
            <Layout>
                <h1>Hello Prezly 👋</h1>
                <PaginatedStories stories={stories} pagination={pagination} />
            </Layout>
        );
    }
    ```

    `PaginatedStories` renders `StoriesList` followed by the `Pagination` component. Users move between pages of results using previous/next controls.

    ```tsx
    // modules/PaginatedStories/PaginatedStories.tsx
    export function PaginatedStories({ stories, pagination }: Props) {
        return (
            <>
                <StoriesList stories={stories} />
                <Pagination {...pagination} />
            </>
        );
    }
    ```
  </Tab>
  <Tab title="InfiniteStories (opt-in)">
    ```tsx
    // modules/Stories/Stories.tsx
    import { InfiniteStories } from '../InfiniteStories';

    export function Stories({ stories, pagination }: Props) {
        return (
            <Layout>
                <h1>Hello Prezly 👋</h1>
                {/* <PaginatedStories stories={stories} pagination={pagination} /> */}
                <InfiniteStories initialStories={stories} pagination={pagination} />
            </Layout>
        );
    }
    ```

    `InfiniteStories` uses the `useInfiniteStoriesLoading` hook from `@prezly/theme-kit-nextjs` to append more stories to the list without a page reload. A **Load more** button appears below the list when additional pages are available.

    ```tsx
    // modules/InfiniteStories/InfiniteStories.tsx
    export function InfiniteStories({ initialStories, pagination, category }: Props) {
        const { canLoadMore, isLoading, loadMoreStories, stories } = useInfiniteStoriesLoading(
            initialStories,
            pagination,
            category,
        );

        return (
            <>
                <StoriesList stories={stories} />
                {canLoadMore && <LoadMore isLoading={isLoading} onLoadMore={loadMoreStories} />}
            </>
        );
    }
    ```

    The `LoadMore` button is disabled while the next page is fetching and its label switches between the `loadMore` and `stateLoading` translation keys automatically.
  </Tab>
</Tabs>

<Note>
The optional `category` prop on `InfiniteStories` scopes loading to a specific category. When rendering the homepage you can omit it; the category page passes the current category automatically.
</Note>

## Individual story page

Each story is served at `/{slug}` via the Next.js dynamic route `pages/[slug].tsx`. The page calls `getStoryPageServerSideProps` to fetch the full story, then passes it to the `Story` module.

```tsx
// pages/[slug].tsx
const StoryPage: NextPage<BasePageProps> = () => {
    const currentStory = useCurrentStory();
    return <Story story={currentStory!} />;
};
```

### Story rendering

The `Story` module renders the story's `title`, `subtitle`, and rich-text `content`. Content rendering depends on the story's `format_version` field.

```tsx
// modules/Story/Story.tsx
export function Story({ story }: Props) {
    const { title, subtitle, content, format_version } = story;

    return (
        <Layout>
            <StorySeo story={story} />
            <article>
                <h2>{title}</h2>
                <h3>{subtitle}</h3>
                {Number(format_version) === FormatVersion.HTML && (
                    <div dangerouslySetInnerHTML={{ __html: content }} />
                )}
                {Number(format_version) >= FormatVersion.SLATEJS && (
                    <ContentRenderer nodes={JSON.parse(content)} />
                )}
            </article>
        </Layout>
    );
}
```

<Note>
`format_version` determines how content is decoded. Legacy stories use `FormatVersion.HTML` and are injected as raw HTML. Newer stories use `FormatVersion.SLATEJS` (or higher) and are parsed as a JSON node tree rendered by `ContentRenderer`. Always check this field before rendering story content.
</Note>

`StorySeo` from `@prezly/theme-kit-nextjs` automatically populates `<head>` metadata — Open Graph tags, canonical URL, and structured data — based on the story object.

## Draft preview

To preview an unpublished story, Prezly uses the `/s/[uuid]` route (provided automatically by `@prezly/theme-kit-nextjs`). This route bypasses slug-based lookup and fetches the story directly by UUID, so you can share a preview link before publishing.

## Routes summary

| Route | Description |
|---|---|
| `/` | Homepage — paginated or infinite story listing |
| `/[slug]` | Individual published story |
| `/s/[uuid]` | Draft preview by UUID |
